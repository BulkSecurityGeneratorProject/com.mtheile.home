MonaStack - Ein Technologiestack
MonaStack ist eine Sammlung von bewährten und gut kombinierbaren Technologien. Der sich daraus ergebende Stack ermöglicht eine schnelle, sichere und saubere Umsetzung komplexer Web-Entwicklungsprojekte.

Mit MonaStack sind folgende Aspekte abgedeckt:
 
  • Continuous Integration und Testing
  • Versioning, Roadmaps, Releases, SCM
  • Authentication
  • Authorization
  • Scalability via Cloud-Deployment (optional)
  • Clear Component Dependencies
  • DTO-Entity-Mapping/Persistence
  • Custom Datatypes on Client-Side
  • Standard Code-Formatter
  • Out-Of-The-Box Functions:
    • CRUD (create, read, update, delete)
    • MVC (all views get updated on change)
    • Paging
    • Querying
    • Import/Export
    • History-Controller
    • File Upload/Download
    • Default CSS-Styles
Das Erfolgsrezept von MonaStack ist die saubere Anwendung einiger weniger, sorgfältig ausgewählter Technologien. Durch sinnvolle Abstraktionen und klare Konventionen werden Best-Practices gefördert.

Fortlaufend findet eine Anpassung des MonaStacks an die Entwicklungen der IT statt.

Aus unserer Sicht ist eine gut gepflegte Standardarchitektur mit Referenzimplementierung die Grundvoraussetzung für eine erfolgreiche, effiziente und skalierbare Softwareentwicklung.

Bei Interesse stelle ich MonaStack sehr gerne vor. Gerne stehe ich Ihnen auch zur Seite, einen für Sie passenden Stack auszuarbeiten.

DynSBatch - Überlegenheit durch Reflection
DynSBatch ist ein Produkt und ermöglicht schnelle und flexible Datenpflege auf dem Kernbanksystem OSPlus der Sparkassen durch Massenaufträge. Über eine komfortable, GWT-basierte Benutzungsoberfläche im Browser können Aufträge eingestellt, überwacht und ausgewertet werden.
Die besondere Herausforderung bestand in der generischen Anbindung der mehr als 1000 fachlichen WebServices der dynamischen SOAP-Schnittstelle (DynS). Dies wurde durch eine Kombination der Java-Reflection-API und Axis erreicht.
Eine weitere Besonderheit ist eine modulare Funktionserweiterung mit fachlichen Bausteinen. Durch die Nutzung des RCP/OSGi-Frameworks können sogenannte Bundles vom Kunden extra gekauft und dem System per Mausklick hinzugefügt werden.
Das Datenmodell wurde mit MagicDraw definiert und schließlich generiert. In diesem MDA-Ansatz (Model Driven Architekture) kam AndroMDA zum Einsatz.
Die Anwendung ist bei mehr als 30 Sparkassen sehr erfolgreich im Einsatz.
  • Vertrag: Anstellung
  • Rolle: Projektleitung
  • Teamgröße: 3
  • Technologien u.a.:
    • OO-Modellierung / UML
    • MagicDraw, AndroMDA
    • Reflection API
    • Axis / WebService / SOAP
    • RCP / OSGi
    • Jetty
BeteiligungsIntegrator - Zuverlässig trotz schwieriger Umgebung
Der BeteiligungsIntegrator läuft als 24/7 Hintergrundjob. Durch ihn werden Daten geschlossener Fonds im Kernbanksystem von Sparkassen aktuell gehalten. Daten mehrerer Anbieter werden über proprietäre Schnittstellen (secured SOAP, JSON) geholt, semantisch vereinheitlicht und in das OSPlus-System übertragen.
Des Weiteren existiert eine Benutzungsoberfläche zur OSPlus-Administration. Mit ihrer Hilfe kann eine Bank sehr komfortable entsprechende Vertragsarten einrichten.
Herausforderung war die Zuverlässigkeit trotz Anbindung an sehr unterschiedliche Schnittstellen mit einer Vielzahl schwer vorhersehbaren Ereignissen und die Datenabstraktion als Grundlage zur Datenübersetzung.
Der Prozess läuft zuverlässig bei mehreren Banken.

  • Rolle: Projektleitung
  • Teamgröße: 4
  • Technologien u.a.:
    • SOAP/JSON
    • SSH
    • DynS/StatS (OSPlus)
DWH4OSPlus - Trenderkennung durch regelmäßige Datenanalyse
DWH4OSPlus ist ein Framework für die Datenauswertung einer Sparkasse. Geschäftsdaten werden in konfigurierbaren Zeitintervallen analysiert und als Kennzahlen aufbereitet. Die Ergebnisse werden in einer Access-DB festgehalten. Durch RCP-Extension-Points (wie bei DynSBatch) können dem System neue Kennzahl-Module hinzugefügt werden. Sparkassenmitarbeiter können durch Modifikation der Access-DB die Kennzahlen-Darstellungen, z.B. als Charts beliebig anpassen.
Die Herausforderung bestand darin, eine laufende Kennzahlengenerierung mit flexiblen Auswertungs- und Darstellungsmöglichkeiten zu bieten. Dies ist durch den programmatischen Zugriff auf Access-MDB-Dateien gelungen. Die gesamte Steuerung findet über den Browser mit einer GWT-Oberfläche statt.
  • Vertrag: Anstellung
  • Technologien u.a.:
    • Access
    • ODBC
    • JDBC
    • SQL (Access, DB2)

SparkassenMigrator - Die Migrationshilfe
Der SparkassenMigrator diente der Aufbereitung von Personendaten zur Migration aus dem Bestandssystem der bayrischen Sparkassen in das neu einzuführende Kernbanksystem OSPlus der Finanz Informatik. Dazu mussten zunächst die in Form großer CSV-Dateien vorliegenden Personendaten anhand eines komplexen Datenmodells in eine Datenbank importiert werden. Anschließend wurden die Daten über eine GWT-Oberfläche im Browser visualisiert und von den Bearbeitern editiert. Die aufbereiteten Daten wurden dann über ein Drittsystem in das Bestandssystem zurückgeschrieben.
Die Hauptanforderungen lagen einerseits in der Bewältigung der komplexen Fachlichkeit, andererseits in der Optimierung des Systems für große Datenmengen und den Mehrbenutzerbetrieb. Dies gelang in weniger als 2,5 Monaten Entwicklungszeit.
Das System kam sehr erfolgreich bei über zwanzig Sparkassen zum Einsatz.

  • Rolle: Projektleitung
  • Teamgröße: 4
  • Technologien u.a.:
    • OO-Modellierung, UML
    • MagicDraw, AndroMDA

WAC-Manager - Verwaltungssoftware für den Wolfratshauser Aktienclub
Monatlich zahlen die Mitglieder des Clubs fast beliebige Beträge in das Konto des WAC ein. Von diesem Kapital wurden dann, durch den Anlageausschuss festgelegte Aktien gekauft. Anfangs war die Aufgabe der Software "nur" die Anteile der Clubmitglieder aus ihren Guthaben und den Depotentwicklungen zu berechnen. Im Laufe der Zeit hat sich der WAC-Manager zu einem Mitgliederportal mit individuellen Chartdarstellungen, Adressverwaltung, etc. entwickelt. Die WAC-Verwaltung konnte Jahresberichte für alle Mitglieder und den Steuerbericht auf Knopfdruck generieren.
Mit dem WAC-Manager wurden über fast 10 Jahre mehrere hunderttausend Euro verwaltet.
2011 wurde der Club aus Zweifel an Börsenspekulationen aufgelöst.
  • Vertragsart: Auftrag
  • Technologien u.a.:
    • JFreeChart
    • JSP (Java Server Pages)
    • Struts
    • XPath/XSD
    • Xerces
